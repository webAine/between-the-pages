## Идеи для геймплея — поиск книг

Не в разработке, записано для обсуждения.

- [ ] **Инспекция книги** — клик не отдаёт сразу. Панель: цвет, толщина, состояние, декор, название (если читаемо). Кнопки: "Отдать" / "Положить обратно".
- [ ] **Названий на полке нет** — только цвет, толщина, состояние, декор. Название — только при инспекции. Каталог становится обязательным для всех запросов.
- [ ] **Состояние влияет на читаемость** — потёртые книги при инспекции: "название неразборчиво". Приходится доверять только физическим признакам.
- [ ] **Несколько изданий одной книги** — две копии с разным видом. Покупатель хочет конкретное издание — ищешь правильный экземпляр по физическим признакам.
- [ ] **Книга не на месте — привязка к Порядку** — процент смещённых книг зависит от стата: 0–30 → ~40%, 31–60 → ~20%, 61–100 → ~5%. Книга не пропадает, просто лежит в другом шкафу/ряду. Порядок ощущается как реальный прогресс.
- [ ] **Продолжение серии** — покупатель хочет книгу 2, имея книгу 1. В описании первой упоминается продолжение.
- [ ] **Книга по настроению** — покупатель описывает атмосферу, не жанр. Читаешь описания в каталоге и сопоставляешь.
- [ ] **Состояние книги как требование** — покупатель хочет новую (подарок) или читаную. На полке отображается состояние.
- [ ] **Сравнительный поиск** — "хочу похожее на [название], но короче". Берёшь признаки из карточки → ищешь по фильтру.
- [ ] **Переименование шкафов** — игрок даёт шкафам свои названия ("Детективы у окна"). Помогает ориентироваться.
